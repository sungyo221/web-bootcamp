📑 5주차 정리 — 비동기 & fetch 🌐

인터넷에서 진짜 데이터를 가져와 앱을 만드는 기술.

1. 동기 vs 비동기

인터넷에서 데이터 받기는 시간이 걸려요(서버 다녀와야 하니까). 그래서 "기다렸다 받는" 비동기가 필요해요.

구분비유
동기물 끓을 때까지 가만히 서서 기다림
비동기물 올려두고 다른 일 하다가, 끓으면 면 넣음

2. fetch 3줄 공식 ⭐

async function 데이터가져오기() {
  const response = await fetch("주소");    // ① 서버에 요청 (기다림)
  const data = await response.json();      // ② 받은 걸 JS로 변환 (기다림)
  console.log(data);                       // ③ 사용!
}
키워드뜻
fetch("주소")그 주소로 데이터 요청
await"기다려라" (결과 올 때까지)
asyncawait를 쓰려면 함수 앞에 필수
.json()받은 데이터를 JS 객체/배열로 변환
⚠️ await는 async function 안에서만 쓸 수 있어요. 둘은 짝꿍!

3. 받은 데이터 꺼내기

API마다 데이터 모양이 달라요! 그래서 "이 API가 뭘 주는지" 먼저 확인해야 해요.

// 객체를 줄 때 → 점(.)으로 속성 꺼내기
data.message     // 강아지 사진 주소
data.fact        // 고양이 상식

// 배열을 줄 때 → 반복문으로 돌기 (3주차!)
for (let i = 0; i < 10; i++) {
  console.log(data[i].title);     // i번째 항목의 title
}
💡 막히면 console.log(data) 를 찍어 구조를 확인! F12 콘솔에서 어떤 모양인지 보여요.

4. try / catch (에러 처리)

인터넷이 끊기거나 문제가 생겨도 앱이 안 멈추게 막는 안전망.

try {
  const response = await fetch("주소");   // 위험한 작업
  const data = await response.json();
  // ...사용
} catch (error) {
  console.log("실패!", error);            // 실패하면 여기로
}
💡 실무 fetch엔 거의 항상 try/catch를 붙여요. "잘 될 때"뿐 아니라 "안 될 때"도 대비!

5. 🧩 검색 앱 패턴 (총정리)

async function search() {
  const keyword = input.value;              // 4주: 입력값
  if (keyword === "") return;               // 4주: 빈칸 방어
  try {
    const res = await fetch(BASE + keyword); // 5주: 검색어를 주소에 +
    const data = await res.json();           // 5주: 변환
    list.innerHTML = "";
    for (let i = 0; i < 10; i++) {           // 3주: 반복
      const li = document.createElement("li");
      li.textContent = data.docs[i].title;   // 3주: DOM + 속성
      list.appendChild(li);
    }
  } catch (e) { /* 5주: 에러 처리 */ }
}
btn.addEventListener("click", search);                    // 3주: 이벤트
input.addEventListener("keydown", e => {                  // 4주: Enter
  if (e.key === "Enter") search();
});
💡 검색의 핵심: 검색어를 주소에 이어붙이기 fetch(주소 + keyword). 모든 검색 기능이 이 원리예요!

  • 배열이면 반복문, 객체면 점(.) 으로 꺼내기
