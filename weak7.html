📑 7주차 정리 — React ⚛️

화면을 컴포넌트(조각)로 만들고, state로 변하는 값을 다루는 라이브러리.

0. 왜 React?

바닐라 JS (3주차)React
화면 그리기createElement+appendChild 직접.map() 한 줄
화면 갱신render() 수동 호출state 바뀌면 자동
💡 핵심 사상: "어떻게 그릴지" 말고 "무엇을 보여줄지"만 정하면 React가 알아서 그려요 (선언적 UI).

1. 컴포넌트 & JSX

// 컴포넌트 = JSX 를 return 하는 함수
function Welcome() {
  return <h1>안녕하세요!</h1>;
}

// 사용: HTML 태그처럼
<Welcome />
⚠️ JSX 3규칙
1) 컴포넌트 이름은 대문자 시작 (Welcome O / welcome X)
2) class 대신 className
3) return 은 태그 하나로 감싸기 (<div> 또는 <>...</>)

2. props (값 전달) 📦

// 보낼 때 (HTML 속성처럼)
<Profile name="정선교" job="개발자" />

// 받을 때 (함수 매개변수처럼)
function Profile(props) {
  return <h2>{props.name} - {props.job}</h2>;
}
💡 props = 컴포넌트에 넘기는 값. 같은 컴포넌트 + 다른 props = 다른 화면 → 재사용! (함수 매개변수와 동일)

3. JSX 안에서 JS 쓰기 — { }

const name = "정선교";
<p>{name}</p>                          // 변수
<p>{2 + 3}</p>                         // 계산
<p>{on ? "켜짐 💡" : "꺼짐 🌑"}</p>     // 삼항연산자(조건)
⚠️ 화면에 변수·조건을 보여주려면 태그 안에 { }로 감싸기! (가장 많이 쓰는 규칙)

4. useState (상태) 🧠 ⭐

import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);   // 0으로 시작
  //      ↑현재값  ↑바꾸는함수      ↑초기값
  return (
    <button onClick={() => setCount(count + 1)}>
      {count}
    </button>
  );
}
⚠️ 값은 꼭 set함수로 바꾸기! setCount(...) O / count = ... X
(set함수를 써야 React가 화면을 다시 그려요)

5. 이벤트 (onClick 등)

<button onClick={handleClick}>클릭</button>          // 함수 이름
<button onClick={() => setCount(count+1)}>+1</button> // 화살표 함수
<input value={text} onChange={(e) => setText(e.target.value)} />  // 입력창
JSX 이벤트언제
onClick클릭
onChange입력값 바뀔 때 (e.target.value)
💡 입력창은 value={state} + onChange 로 state와 연결 (controlled input).

6. 목록 렌더링 .map() 🔁

{todos.map((todo, index) => (
  <li key={index}>{todo}</li>
))}
💡 .map() = 배열의 각 항목을 화면 요소로 변환 (3주차 for의 React 버전). key는 각 항목 구분용 (필수!).

배열 state 바꾸기 (불변성!)

setTodos([...todos, newItem]);              // 추가 (기존 + 새것)
setTodos(todos.filter((_, i) => i !== idx)); // 삭제 (그것만 빼고)
⚠️ React에선 push 대신 새 배열을 만들어 set함수로 교체! [...todos, x] 의 ...(점3개)는 "기존 항목 펼치기".

7. 개발 환경 (Vite)

cd my-app

✅ 체크리스트

  • 컴포넌트 이름 대문자 / className / return은 태그 하나로
  • 화면에 값 보이려면 태그 안 { }
  • state는 useState, 바꿀 땐 set함수 (직접 대입 X)
  • 목록은 .map() + key
  • 배열 state는 새 배열로 교체 ([...todos, x])
