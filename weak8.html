📑 8주차 정리 — 컴포넌트 분리 & 목록 📂📋

컴포넌트를 파일로 나누고, 데이터로 화면을 만드는 진짜 React 구조.

1. 컴포넌트 파일 분리 (export / import)

컴포넌트마다 파일 하나씩! 큰 앱을 깔끔하게 관리해요.

// components/Card.jsx — 만들어서 '내보내기'
function Card(props) {
  return <div>{props.title}</div>;
}
export default Card;          // ★ 이 파일의 Card 를 밖에서 쓸 수 있게

// App.jsx — '가져오기'
import Card from "./components/Card";   // ★ Card 불러오기
키워드뜻
export default 이름이 컴포넌트를 내보냄 (파일당 1개)
import 이름 from "경로"다른 파일에서 가져옴
⚠️ 둘은 짝! 내보내기(export default) 안 하면 가져올 수 없어요. 경로의 ./ 는 "같은 폴더 기준"이에요.

2. 폴더 구조

src/
├── App.jsx              ← 조립하는 곳
└── components/
    ├── Card.jsx         ← 카드 부품
    └── Footer.jsx       ← 푸터 부품
💡 레고처럼 부품(컴포넌트 파일)을 만들어서 App에서 조립해요. 파일이 100개가 돼도 원리는 똑같아요.

3. 데이터 배열 → 컴포넌트 목록 ⭐ (React의 꽃!)

// 데이터 배열 (객체들의 배열)
const skills = [
  { emoji: "🌐", title: "HTML", desc: "뼈대" },
  { emoji: "🎨", title: "CSS",  desc: "디자인" },
];

// .map() 으로 각 데이터를 컴포넌트로! props 로 값 전달
{skills.map((skill, index) => (
  <Card
    key={index}
    emoji={skill.emoji}
    title={skill.title}
    desc={skill.desc}
  />
))}
부분뜻
skills.map((skill, i) => ...)배열의 각 객체 → 컴포넌트 1개
emoji={skill.emoji}그 객체의 값을 props 로 전달
key={index}각 항목 구분 (필수!)
💡 데이터만 바꾸면 화면이 따라와요! 배열에 항목 추가 → 카드 자동 1개 더. 인스타 피드·유튜브·쇼핑몰 전부 이 패턴.

🧩 전체 그림

데이터(배열)  →  .map()  →  <Card /> 여러 개  (props로 값 주입)
   ↑
컴포넌트는 다른 파일에서 import 해옴
💡 "어떤 데이터를 보여줄지"만 정하면, React가 "어떻게 그릴지"를 알아서 해요 (선언적 UI).

✅ 체크리스트

  • 컴포넌트 파일 끝에 export default 이름
  • 쓰는 파일 위에 import 이름 from "./경로"
  • 데이터는 객체들의 배열로
  • 목록은 .map() + key
  • 각 값은 props로 컴포넌트에 전달
