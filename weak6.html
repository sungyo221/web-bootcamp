1. Git 핵심 흐름 (add → commit → push) ⭐

git add .                    # ① 변경 파일 담기 (장바구니에 담기)
git commit -m "메모"          # ② 세이브 + 설명 남기기 (세이브 포인트!)
git push                     # ③ GitHub에 업로드 (클라우드 동기화)
명령어뜻
git add .모든 변경 파일 담기 (마침표 = 전부)
git commit -m "메모"세이브 + 무엇을 바꿨는지 설명
git pushGitHub(인터넷)에 올리기
💡 앞으로 코드를 바꿀 때마다 이 3줄이면 끝! push하면 1~2분 뒤 배포 사이트에도 자동 반영돼요.

2. 저장소 처음 만들 때 (최초 1회만)

git init                     # 이 폴더를 Git 저장소로 만들기
git add .                    # 전부 담기
git commit -m "첫 커밋"       # 첫 세이브

# GitHub에서 빈 저장소 만든 뒤, 연결하고 업로드:
git remote add origin https://github.com/아이디/저장소.git
git branch -M main           # 기본 브랜치 이름을 main 으로
git push -u origin main      # 첫 업로드 (브라우저 로그인 1회)
⚠️ GitHub에서 저장소 만들 때 README·.gitignore 등은 체크하지 말기 (이미 로컬에 파일이 있으면 충돌나요). 처음 push 때 브라우저 로그인이 한 번 떠요 (그 뒤론 자동).

3. GitHub Pages 배포 (웹사이트 공개)

  1. 저장소를 Public(공개)으로 (Settings → 맨 아래 Danger Zone → Change visibility)
  2. Settings → Pages
  3. Source: Deploy from a branch / Branch: main / 폴더: / (root) → Save
  4. 1~2분 뒤 주소가 생김!
⚠️ 무료 배포는 저장소가 Public이어야 해요. 학습 코드는 공개해도 안전 (= 포트폴리오!).
💡 사이트 주소 규칙: https://아이디.github.io/저장소이름/

✅ 체크리스트 & 자주 쓰는 명령

  • 코드 바꾸면 → git add . → git commit -m "메모" → git push
  • 현재 상태 확인: git status
  • 지금까지 기록 보기: git log --oneline
  • commit 메모는 무엇을 왜 바꿨는지 알아보게 쓰기
  • 배포 사이트는 push 후 1~2분 뒤 반영 (안 보이면 새로고침)
  • GitHub 잔디(커밋 기록) = 이력서·포트폴리오가 됨!
