요청을 받아 응답을 주는 서버를 직접 만들고, 프론트와 연결(CRUD).

0. 프론트 vs 백엔드

프론트엔드백엔드
어디브라우저 (localhost:5173)서버 (localhost:3000)
역할화면 보여주기 (React)데이터 주고받기 (Express)
[브라우저]  ──요청(request)──▶  [서버]
 (프론트)   ◀──응답(response)──   (백엔드)

1. Express 서버 기본

import express from "express";
const app = express();

app.get("/", (req, res) => {      // "/" 로 오면
  res.send("안녕!");               // 이걸 응답
});

app.listen(3000, () => {          // 3000번 포트에서 켜기
  console.log("서버 실행 중!");
});
코드뜻
app.get("주소", (req,res)=>...)그 주소로 오면 실행할 함수 (라우트)
res.send("글자")글자 응답
res.json({...})JSON 데이터 응답 (fetch로 받는 그것!)
app.listen(3000)서버 켜기
⚠️ 서버는 자동 새로고침 X. 코드 바꾸면 재시작 필요 → node --watch server.js 로 켜면 자동 재시작!

2. CRUD — 4가지 요청 방식 ⭐

글자뜻방식(method)
C reate추가app.post
R ead읽기app.get
U pdate수정app.put
D elete삭제app.delete
let products = [ { id: 1, name: "키보드", price: 35000 } ];

app.get("/api/products", (req, res) => res.json(products));          // 읽기

app.post("/api/products", (req, res) => {                            // 추가
  const item = { id: Date.now(), name: req.body.name };
  products.push(item);
  res.json(item);
});

app.delete("/api/products/:id", (req, res) => {                      // 삭제
  const id = Number(req.params.id);
  products = products.filter((p) => p.id !== id);
  res.json({ ok: true });
});
코드뜻
req.body프론트가 보낸 데이터 (POST) — app.use(express.json()) 필요
:id / req.params.id주소에 담긴 값 (예: /products/3)

3. React ↔ 서버 연결 (fetch) 🔗

읽기 (GET) — useEffect 안에서

useEffect(() => {
  async function load() {
    const res = await fetch("http://localhost:3000/api/products");
    const data = await res.json();
    setProducts(data);
  }
  load();
}, []);   // ← 빈 배열 = 처음 한 번만

추가 (POST) — 옵션 붙여서

await fetch(API, {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ name, price }),   // 보낼 데이터를 글자로 포장
});

삭제 (DELETE) — 주소에 id

await fetch(`${API}/${id}`, { method: "DELETE" });
⚠️ CORS: 프론트(5173)와 서버(3000)는 주소가 달라서, 서버가 Access-Control-Allow-Origin 헤더로 허용해줘야 브라우저가 안 막아요.
💡 useEffect = 컴포넌트가 처음 뜰 때 한 번 실행 (데이터 불러오기용). useState와 함께 React 필수 훅!

🧭 실무 개발 순서 (기능 단위!)

백엔드 전부 → 프론트 전부가 아니라, 기능 하나를 세로로 완성:

기능 1개 ("상품 추가")를 만들 때:
  ① 데이터 구조 정하기
  ② 백엔드 API 만들기 + 테스트 (curl 등)
  ③ 프론트 화면 만들기
  ④ fetch 로 연결 + 눌러서 확인
  ⑤ git commit & push
→ 다음 기능(삭제)도 ①~⑤ 반복
💡 핵심: 작게, 하나씩 / 자주 커밋 / API 먼저 테스트 (프론트 붙이기 전에).

✅ 체크리스트 & 실행법

  • 서버 켜기: cd server && node --watch server.js → localhost:3000
  • POST 데이터 읽으려면 app.use(express.json()) 필수
  • 다른 주소에서 부르면 CORS 허용 헤더 필요
  • JSON 응답은 res.json(...), 글자는 res.send(...)
  • 삭제/수정은 서버 + 화면 둘 다 갱신
  • 서버 데이터는 재시작하면 초기화 (진짜 저장은 DB 필요 — 다음에!)
